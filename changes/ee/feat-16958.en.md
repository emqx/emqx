Added focused `/api-spec` endpoints and a dashboard API spec explorer to support
drill-down discovery of EMQX HTTP API context, especially for AI agents and
other tools that benefit from fetching only the relevant API slices instead of a
single bloated spec.

EMQX now serves tag-scoped and drill-down OpenAPI slices. This helps
AI-assisted workflows stay focused by narrowing the retrieved context to the
relevant API area. These endpoints are disabled together with Swagger when
`dashboard.swagger_support` is set to `false`.

Added `emqx ctl api_keys` CLI commands to list, show, add, delete, enable, and disable API keys from the command line.
