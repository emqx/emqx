Strengthen dashboard administrator password and API key secret hashing.

New dashboard passwords use PBKDF2-HMAC-SHA256 with 600,000 iterations
and a 16-byte random salt, replacing the previous single-iteration
SHA-256 with a 2-byte salt.

API key secrets are also rehashed with a 16-byte random salt in the
same self-describing storage format, but without iteration stretching:
API secrets are 32+ bytes of random data, so a slow KDF would add
material per-request CPU cost (~500ms per HTTP API call) without
strengthening already-uncrackable secrets.

Existing stored hashes continue to verify; they are rewritten in the
new format the next time the user changes their password (admin
passwords) or the credential is recreated (API keys).
