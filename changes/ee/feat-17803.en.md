When EMQX is booted with `EMQX_FEATURES=ESSENTIAL`, the Erlang code loading mode now defaults to `interactive` so that the `.beam` files of disabled features are loaded on demand instead of all at boot. This significantly reduces the resident memory footprint of an essential-mode node, since the modules belonging to skipped features never become resident. The mode can still be overridden by setting `CODE_LOADING_MODE` explicitly.
