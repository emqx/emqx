Upgraded Kafka client libraries: `brod` from 4.5.2 to 4.5.4 and `wolff` from 4.1.9 to 4.1.10.

Notable fixes picked up from upstream:
- `brod`: fix a race condition during Kafka connection re-authentication (via `kafka_protocol` 4.3.4).
- `brod`: fix aborted-transaction filtering when consuming with `read_committed` isolation level.
- `brod`: stop silently falling back to the default begin offset when Kafka's `OffsetFetch` returns -1; the fallback offset is now resolved explicitly from the consumer configuration and logged.
- `wolff`: under high-memory load control (`drop_if_highmem`), keep a minimum buffer reserve so the producer is not starved of in-flight data; only bytes exceeding the reserve are dropped.
