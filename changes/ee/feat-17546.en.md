Added `mqtt.max_session_expiry_interval` config to cap the session expiry interval an MQTT 5.0 client may request via its `Session-Expiry-Interval` property. If the client requests longer than this limit, the server silently clamps it and reflects the clamped value back in the CONNACK. Defaults to `infinity` (no clamp), preserving existing behavior. Has no effect on MQTT 3.1.1/3.1 clients, whose session expiry remains fully server-controlled via `mqtt.session_expiry_interval`.
