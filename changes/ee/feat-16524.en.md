Enhanced base64 encoding and decoding functions with support for padding and URL-safe options.

Added new function variants `base64_encode/2`, `base64_encode/3`, `base64_decode/2`, and `base64_decode/3` that support the following options:

- `'no_padding'`: Encode/decode without padding characters (`=`)
- `'urlsafe'`: Use URL-safe base64 encoding/decoding (replaces `+` with `-`, `/` with `_`)

The options can be specified in any order when using the 3-argument variants. The `base64_decode/2` and `base64_decode/3` functions default to `#{padding => false}`, allowing them to decode both padded and unpadded base64 strings.

Examples:
- `base64_encode(Data, <<"no_padding">>)` - Encode without padding
- `base64_encode(Data, <<"urlsafe">>)` - Encode with URL-safe characters
- `base64_encode(Data, <<"no_padding">>, <<"urlsafe">>)` - Encode with both options
- `base64_decode(Encoded, <<"urlsafe">>)` - Decode URL-safe base64
- `base64_decode(Encoded, <<"urlsafe">>, <<"no_padding">>)` - Decode with both options
