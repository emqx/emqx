Fixed the `actions.executed` metric undercounting action invocations in non-batch mode (`batch_size = 1`).

The previous implementation incremented `actions.executed` from the buffer-worker telemetry flush handler, which collapses many per-message completions into one event, so the counter fell behind the actual number of action callback invocations even when no batching was configured.

`actions.executed` is now incremented once per action callback invocation (one per batch in batch mode, one per message in single mode), independently of the telemetry flush window.
