Fixed the ACME plugin's `acc_key_password` setting. The configured `file://` URI was being passed verbatim to the PEM decoder as the literal password, so an encrypted ACME account-key PEM never decrypted. The plugin now reads the password file (trimming a trailing newline) and hands the actual password to the ACME client. The field also accepts an inline password string for ops who prefer that over a separate file — `file://<path>` selects file mode, anything else is used verbatim.
