Fixed a crash in the rule engine `$events/message/delivered` and `$events/message/acked` events when a message was delivered through a durable session. The session's storage layer used to drop the message ID before serialization and never restored it, so rules selecting the `id` column crashed with `function_clause` in `emqx_guid:to_hexstr/1`. The ID is now preserved end-to-end, and the event handlers tolerate empty IDs from messages that were already on disk before the upgrade.
